# 🏭 妍發科技 製程管理系統

## 一、快速啟動

```bash
# 1. 進入專案目錄
cd 妍發科技/yenfa_system

# 2. 啟動虛擬環境
source ../../.venv/bin/activate

# 3. 啟動系統
streamlit run app.py
```

瀏覽器會自動打開 **http://localhost:8501**

---

## 二、讓其他人也能使用

### 方法 1️⃣：區域網路共享（最簡單，適合辦公室內）

在你的電腦上執行：

```bash
# 查詢你的 IP 位址
ifconfig | grep "inet " | grep -v 127.0.0.1

# 啟動系統（對外開放）
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

然後告訴同事在瀏覽器打開：
```
http://你的IP:8501
例如：http://192.168.1.100:8501
```

**優點**：零成本、立即可用  
**限制**：你的電腦必須開機、同事必須在同一個網路

---

### 方法 2️⃣：Streamlit Community Cloud（免費雲端部署）

1. 把 `yenfa_system/` 推上 GitHub（私有 repo 也可以）
2. 到 [share.streamlit.io](https://share.streamlit.io) 用 GitHub 帳號登入
3. 選擇 repo → 指定 `app.py` → Deploy
4. 取得一個公開 URL，任何人都能用

需要加一個 `requirements.txt`：
```
streamlit
pandas
xlrd
```

**優點**：免費、24小時在線、任何地方都能用  
**限制**：資料在雲端（需注意機密性）

---

### 方法 3️⃣：部署到伺服器（正式營運）

如果公司有伺服器或想租雲端 VPS：

```bash
# 在伺服器上
git clone <你的repo>
cd yenfa_system
pip install -r requirements.txt
python import_xls.py          # 匯入資料
nohup streamlit run app.py --server.address 0.0.0.0 --server.port 80 &
```

可搭配 Nginx 反向代理 + HTTPS 做更安全的部署。

## 三、系統操作指南

### 📊 儀表板（首頁）

打開系統就會看到的總覽頁面：

| 區塊 | 說明 |
|------|------|
| 上方指標卡 | 訂單總數、製程中、已出貨、延遲、暫停的即時數字 |
| 金額統計 | 總金額 / 已出貨金額 / 未出貨金額 |
| 訂單狀態分布 | 長條圖顯示各狀態的訂單數量 |
| 廠商負載排行 | 表格列出每個廠商的製程數量與完成率 |
| 月出貨趨勢 | 折線圖顯示每月出貨金額走勢 |

→ **用途**：老闆/業務快速掌握全局

---

### 📋 訂單管理

| 操作 | 說明 |
|------|------|
| 篩選 | 上方可依「狀態」下拉篩選，或輸入「料號」搜尋 |
| 瀏覽 | 表格顯示所有訂單，**狀態會自動標色**（綠=出貨、黃=製程中、紅=延遲） |
| 新增訂單 | 展開「➕ 新增訂單」，填入料號、數量、金額、廠商、交期 |
| 更新狀態 | 展開「✏️ 更新訂單狀態」，輸入訂單 ID 和新狀態 |

→ **用途**：取代「出貨排程 Excel」的手動管理

---

### 🔧 製程追蹤

| 操作 | 說明 |
|------|------|
| 搜尋料號 | 輸入料號（部分也行），顯示該料號的整條製程鏈 |
| 進度條 | 每個料號上方有進度條，一目了然完成幾道製程 |
| 製程明細 | 表格列出每道製程的廠商、預定日期、實際日期、狀態 |
| 全部總覽 | 不輸入料號時，顯示所有料號的完成率排行 |
| 新增製程 | 展開「➕ 新增製程步驟」，可新增製程和排程 |
| 更新製程 | 展開「✏️ 更新製程步驟」，回報實際完成日期 |

→ **用途**：取代「急件表 Excel」的 179 欄橫向追蹤

---

### 🚚 出貨排程

| 操作 | 說明 |
|------|------|
| 搜尋 | 依料號或月份 (YYYY-MM) 篩選出貨記錄 |
| 統計 | 下方自動顯示筆數、總數量、總金額 |
| 登記出貨 | 切到「➕ 新增出貨」頁籤，填入料號、數量、日期 |

→ **用途**：記錄實際出貨，系統自動更新訂單狀態

---

### 🏗️ 廠商統計

| 操作 | 說明 |
|------|------|
| 負載統計 | 表格 + 長條圖顯示每個廠商的總製程數、已完成、進行中、延遲 |
| 廠商清單 | 列出所有廠商的聯絡人、電話 |
| 新增廠商 | 展開表單新增或更新廠商資訊 |

→ **用途**：掌握哪些廠商負載重、哪些延遲多

---

### 📥 資料匯入

| 操作 | 說明 |
|------|------|
| 匯入製程追蹤 | 將 `2026.03急件.xls` 的資料匯入系統 |
| 匯入出貨排程 | 將 `2026當周出貨排程.xls` 的資料匯入系統 |
| 完整重新匯入 | 清空資料庫後從兩個 XLS 重新匯入 |

→ **用途**：第一次使用時匯入舊資料，之後新資料直接在系統中輸入

## 四、日常工作流程建議

```
早上（廠長）：
  1. 打開「🔧 製程追蹤」
  2. 搜尋今天要跟催的料號
  3. 確認各製程進度，回報實際完成日期

中午（業務）：
  1. 打開「📋 訂單管理」
  2. 篩選「製程中」查看哪些還沒出
  3. 有出貨的到「🚚 出貨排程」登記

週末/月底（主管）：
  1. 打開「📊 儀表板」看整體數字
  2. 看「🏗️ 廠商統計」找出延遲最多的廠商
  3. 月出貨趨勢判斷營業額走向
```

## 五、技術架構

```
前端: Streamlit (Python Web Framework)
後端: SQLite 資料庫 (yenfa.db)
資料表:
  - orders         訂單主表
  - process_steps  製程追蹤表
  - shipments      出貨記錄表
  - vendors        廠商主表
```

所有資料存在 `yenfa_system/yenfa.db` 這一個檔案中，備份只需複製這個檔案。
